1. Создать 2 папки для клиентской и серверной части (client_dist и server_dist)
2. Добавить в каждую папку файл setup.py (разница только в параметрах name="***_client" или
    name="***_server" и description="***_client" или description="***_server")
3. Добавить в папку код ПАКЕТ проекта для клиентской и серверной частей соответственно.
4. +Убедиться в том, что установлен пакет wheel (pip install wheel если не установлен)
5. Создать дистрибутивы для клиентской и серверной частей с помощью команды:
    python setup.py sdist bdist_wheel
6. +Создаём аккаунт на https://pypi.org/ (подтверждение регистрации может занять до часа)
7. Создаём файл ключей .pypirc и сохраняем в домашней папке (туда, где хранятся .ssh-ключи)
    (перейти в домашнюю папку можно командой $ cd ~)
8. Загрузить дистрибутив на https://pypi.org/ можно командой:
    twine upload dist/* --verbose


----------------------------------------------------------------------------
Для создания исполняемого файла server_run необходимо:
- в setup.py добавить параметр
        scripts=['server/server_run']
- добавить в проект сам исполняемый файл server_run
